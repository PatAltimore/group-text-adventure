## Task

Investigated and fixed "Failed to connect: Negotiate failed: 404" on deployed app.

## Root Cause

`api/package.json` had `"main": "src/functions/*.js"` — a glob pattern. Azure Functions v4 worker resolves globs to discover function files. However, `glob` module is only a transitive devDependency (via Jest → @jest/core → @jest/reporters → glob). Production deploy runs `npm install --omit=dev`, stripping `glob` from the deployment zip. Without glob resolution, the worker finds 0 functions and returns 404.

## What Changed

**New file:** `api/src/index.js`
- Explicitly imports `./functions/negotiate.js` and `./functions/gameHub.js`
- Deterministic — no glob resolution needed

**File:** `api/package.json` (line 5)
- Changed `"main"` from `"src/functions/*.js"` to `"src/index.js"`

## Rationale

Glob patterns in `package.json` `main` field are fragile in Azure Functions v4. Glob resolution depends on runtime/worker version and package availability. An explicit entry point that imports all function registration modules is deterministic and production-safe.

## Testing

All 111 tests still pass. Function registration verified in `api/src/index.js`.

## Key Learning Documented

**Learning:** Azure Functions v4 main field should never use glob patterns. Use an explicit entry point file.
