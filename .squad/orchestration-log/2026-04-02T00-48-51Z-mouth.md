## Assignment

Fix PowerShell 5.1 compatibility issue in `deploy/deploy.ps1` where `Join-Path` was using 3-4 positional arguments (only available in PowerShell 7+).

## Problem

The deployment script uses Windows PowerShell 5.1 on CI/CD systems and build agents. The `Join-Path` cmdlet in Windows PowerShell 5.1 only accepts 2 positional parameters, but the script was calling it with 3-4 parameters on lines 445-448, causing syntax errors.

## Solution Applied

Refactored all 4 `Join-Path` calls to use nested invocations with 2 positional parameters each instead of chaining multiple paths in a single call:

**Before (PS 7+ only):**
```powershell
$path = Join-Path $dir "api" "src" "functions"
```

**After (PS 5.1 compatible):**
```powershell
$path = Join-Path (Join-Path $dir "api") "src"
$path = Join-Path $path "functions"
```
